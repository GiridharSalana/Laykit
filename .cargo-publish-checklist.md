# Cargo Publish Checklist

Before publishing to crates.io, ensure:

## Required
- [ ] Update repository URL in Cargo.toml
- [ ] Verify all tests pass: `cargo test`
- [ ] Check formatting: `cargo fmt -- --check`
- [ ] Run clippy: `cargo clippy -- -D warnings`
- [ ] Build documentation: `cargo doc --no-deps`
- [ ] Test on clean environment
- [ ] Update CHANGELOG.md with release notes
- [ ] Tag release in git: `git tag -a v0.1.0 -m "Release v0.1.0"`

## Optional but Recommended
- [ ] Add repository badges to README
- [ ] Set up CI/CD (use .github-ci-example.yml)
- [ ] Create GitHub release notes
- [ ] Add examples to repository
- [ ] Document breaking changes (if any)

## Publishing
```bash
# Dry run first
cargo publish --dry-run

# If successful, publish for real
cargo publish
```

## Post-Publish
- [ ] Verify on crates.io
- [ ] Test installation: `cargo install gds_oas`
- [ ] Update documentation site (if applicable)
- [ ] Announce release
